## Введение

::: {custom-style="Normal"}
Данная работа посвящена разработке функциональной модели информационной системы SportRent — системы управления прокатом спортивного оборудования. Система предназначена для автоматизации процессов аренды спортивного инвентаря, обеспечения прозрачности учета оборудования, упрощения взаимодействия между клиентами и администрацией, а также повышения эффективности управления ресурсами.
:::

::: {custom-style="Normal"}
SportRent представляет собой комплексную информационную систему, охватывающую все аспекты бизнес-процессов проката спортивного оборудования: от онлайн-бронирования и оформления заказов до складского учета, обработки платежей и формирования аналитических отчетов.
:::

::: {custom-style="Normal"}
Для моделирования функциональной структуры системы применяется методология SADT (Structured Analysis and Design Technique) с использованием нотации IDEF0, которая позволяет наглядно представить процессы, информационные потоки и ресурсы, участвующие в работе системы.
:::

\newpage

## Цель создания информационной системы

::: {custom-style="Normal"}
Цель создания ИС SportRent: Автоматизировать управление прокатом спортивного оборудования для повышения эффективности бизнес-процессов, улучшения качества обслуживания клиентов и обеспечения прозрачного контроля над состоянием и доступностью оборудования.
:::

### Обоснование соответствия определению ИС

::: {custom-style="Normal"}
Система SportRent полностью соответствует определению информационной системы, поскольку представляет собой совокупность организационных и технических средств, предназначенных для:
:::

::: {custom-style="Normal"}
- Сбора информации — регистрация заявок клиентов, данных об оборудовании, платежной информации
- Хранения информации — организация базы данных клиентов, каталога оборудования, истории заказов
- Обработки информации — формирование договоров, расчет стоимости аренды, контроль сроков
- Передачи информации — взаимодействие между различными модулями системы и пользователями
- Представления информации — генерация отчетов, вывод статистики, уведомления пользователей
:::

::: {custom-style="Normal"}
Система использует современные информационные технологии и обеспечивает поддержку принятия управленческих решений на основе актуальных данных.
:::

## Краткое описание информационной системы

### Функциональные возможности системы SportRent

::: {custom-style="Normal"}
Система SportRent обеспечивает комплексную автоматизацию всех бизнес-процессов проката спортивного оборудования и включает следующие основные функциональные возможности:
:::

::: {custom-style="Normal"}
1. Управление каталогом оборудования — ведение полного каталога спортивного инвентаря с характеристиками, фотографиями и ценами. Категоризация оборудования. Управление ценообразованием и тарифными планами. Контроль технического состояния оборудования.
:::

::: {custom-style="Normal"}
2. Бронирование и заказы — онлайн-бронирование оборудования с выбором даты и времени. Проверка доступности инвентаря в реальном времени. Формирование и обработка заказов. Управление очередью бронирований и резервирований.
:::

::: {custom-style="Normal"}
3. Обработка платежей — интеграция с платежными системами для онлайн-оплаты. Расчет стоимости аренды с учетом длительности и тарифов. Формирование и печать квитанций об оплате. Учет депозитов и залоговых платежей.
:::

::: {custom-style="Normal"}
4. Управление клиентами — регистрация и аутентификация пользователей. Управление профилями клиентов. История аренды и статистика использования. Программа лояльности и скидки для постоянных клиентов.
:::

::: {custom-style="Normal"}
5. Складской учет — отслеживание местонахождения и статуса каждой единицы оборудования. Учет выдачи и возврата инвентаря. Контроль технического состояния и необходимости обслуживания. Планирование графика технического обслуживания.
:::

::: {custom-style="Normal"}
6. Операции с базой данных — надежное хранение всех данных системы. Обеспечение целостности и консистентности информации. Резервное копирование и восстановление данных. Оптимизация запросов для быстрого доступа к информации.
:::

::: {custom-style="Normal"}
7. Отчетность и аналитика — формирование отчетов о загрузке оборудования. Финансовая отчетность и анализ доходов. Статистика популярности различных категорий инвентаря. Прогнозирование спроса и планирование закупок.
:::

::: {custom-style="Normal"}
8. Системное администрирование — управление пользователями и правами доступа. Настройка параметров системы. Мониторинг работоспособности и производительности. Управление безопасностью и аудит действий пользователей.
:::

### Архитектурные особенности

::: {custom-style="Normal"}
Система построена на модульной архитектуре с использованием паттерна архитектуры в виде модульной структуры с 8 независимыми, но интегрированными модулями. API реализован в стиле RESTful для взаимодействия клиент-сервер с обменом данными в формате JSON. База данных — реляционная СУБД с нормализованной схемой данных. Протокол связи — HTTPS/TLS для безопасной передачи данных.
:::

### Показатели качества

::: {custom-style="Normal"}
Система разработана с учетом следующих требований к качеству: доступность 99.5% в рабочее время, время отклика максимум 5 секунд для операций бронирования, производительность API 100 запросов в минуту с временем отклика менее 2 секунд, поддержка 100+ одновременных пользователей без деградации производительности. Безопасность обеспечивается шифрованием данных HTTPS/TLS, защитой от SQL-инъекций и XSS-атак.
:::

## Способ создания информационной системы

::: {custom-style="Normal"}
Для разработки информационной системы SportRent выбран метод последовательных приближений (итеративная модель).
:::

### Обоснование выбора методологии

::: {custom-style="Normal"}
Метод последовательных приближений предполагает пошаговую разработку системы с постепенным уточнением требований и улучшением функциональности. Этот подход оптимален для создания ИС SportRent по следующим причинам:
:::

::: {custom-style="Normal"}
- Гибкость требований — возможность корректировки функциональности на основе обратной связи от пользователей
- Снижение рисков — раннее выявление проблем и их устранение на начальных итерациях
- Поэтапная реализация — возможность ввода в эксплуатацию отдельных модулей по мере готовности
- Адаптация к изменениям — быстрое реагирование на изменения бизнес-требований
:::

### Связь с методологией SADT и нотацией IDEF0

::: {custom-style="Normal"}
Методология SADT используется на этапе анализа и проектирования для декомпозиции системы, визуализации процессов, выявления взаимосвязей и документирования требований. Нотация IDEF0 обеспечивает стандартизированный способ представления функциональной модели, что упрощает коммуникацию между всеми участниками проекта и позволяет однозначно интерпретировать требования к системе.
:::

### Этапы разработки

::: {custom-style="Normal"}
В соответствии с методом последовательных приближений, разработка системы включает следующие этапы: исследование и технико-экономическое обоснование, сбор и анализ требований, эскизное проектирование, техническое проектирование, разработка, интеграционное тестирование, опытная эксплуатация и промышленное внедрение.
:::

## Средства создания информационной системы

::: {custom-style="Normal"}
Для разработки и функционирования информационной системы SportRent используется следующий технологический стек:
:::

### Frontend (клиентская часть)

::: {custom-style="Normal"}
HTML5 — разметка структуры веб-страниц. CSS3 — стилизация и адаптивный дизайн интерфейса. JavaScript фреймворки: React, Vue.js или Angular — для создания динамичного, отзывчивого пользовательского интерфейса. Выбор конкретного фреймворка определяется на этапе технического проектирования.
:::

### Backend (серверная часть)

::: {custom-style="Normal"}
Серверная логика может быть реализована на одном из следующих языков: Python (Django, Flask, FastAPI) — высокая скорость разработки, богатая экосистема. Node.js (Express.js) — асинхронная обработка, единый язык с frontend. Java (Spring Boot) — масштабируемость, надежность для enterprise-решений. C# (ASP.NET Core) — высокая производительность, интеграция с Microsoft-технологиями.
:::

### База данных

::: {custom-style="Normal"}
PostgreSQL (рекомендуется) — мощная открытая СУБД с поддержкой ACID и расширенными возможностями. MySQL (альтернатива) — популярная реляционная СУБД с хорошей производительностью. Особенности проектирования БД: нормализованная схема данных (3NF), индексация для оптимизации запросов, механизмы транзакций для обеспечения целостности данных.
:::

### Веб-сервер

::: {custom-style="Normal"}
Nginx 1.18+ (рекомендуется) — высокопроизводительный HTTP-сервер и обратный прокси. Apache 2.4+ (альтернатива) — универсальный веб-сервер с богатой функциональностью.
:::

### Инструменты моделирования

::: {custom-style="Normal"}
Ramus Educational — CASE-средство для создания диаграмм IDEF0 в соответствии с РД IDEF0-2000. PlantUML — инструмент для создания UML-диаграмм из текстового описания.
:::

### Дополнительные технологии

::: {custom-style="Normal"}
RESTful API — архитектурный стиль для взаимодействия клиента и сервера. JSON — формат обмена данными. JWT (JSON Web Tokens) — механизм аутентификации и авторизации. HTTPS/TLS — протокол безопасной передачи данных. Git — система контроля версий для управления исходным кодом. Интеграция с платежными системами (Stripe, PayPal, ЮKassa). Системы уведомлений — email-рассылка, SMS-уведомления.
:::

## Проектирование контекстной диаграммы функциональной модели ИС

### Описание контекстной диаграммы A-0

::: {custom-style="Normal"}
Контекстная диаграмма A-0 представляет систему SportRent на самом верхнем уровне абстракции как единый функциональный блок, взаимодействующий с внешним окружением.
:::

### Функциональный блок A0

::: {custom-style="Normal"}
Название: Автоматизировать управление прокатом спортивного оборудования. Номер: A0.
:::

::: {custom-style="Normal"}
Описание: Центральный функциональный блок представляет всю информационную систему SportRent как единое целое. Он отражает главную цель системы — автоматизацию всех процессов, связанных с прокатом спортивного оборудования, от момента получения заявки клиента до формирования отчетов и аналитики.
:::

::: {custom-style="Normal"}
Блок выполняет следующие основные функции: прием и обработка заявок клиентов на аренду, управление каталогом оборудования и контроль доступности, обработка платежей и финансовых операций, формирование договорной и отчетной документации, учет движения оборудования на складе, предоставление информации о статусе заказов.
:::

### Элементы управления (Control)

::: {custom-style="Normal"}
Элементы управления определяют правила, регламенты и алгоритмы, которым должна следовать система при выполнении своих функций.
:::

::: {custom-style="Normal"}
C1: ФЗ-152 о персональных данных — Федеральный закон Российской Федерации № 152-ФЗ «О персональных данных» регулирует обработку персональных данных клиентов. Система обрабатывает персональные данные клиентов (ФИО, контактные данные, паспортные данные), поэтому обязана соответствовать требованиям законодательства о защите персональных данных.
:::

::: {custom-style="Normal"}
C2: Политика компании — внутренние правила и политики компании SportRent, определяющие бизнес-процессы, тарифы, условия аренды, программы лояльности. Политика компании определяет уникальные бизнес-правила: размер залога, правила возврата оборудования, условия отмены бронирования, скидки для постоянных клиентов.
:::

::: {custom-style="Normal"}
C3: ГОСТ 34.602-2020 — государственный стандарт «Информационные технологии. Комплекс стандартов на автоматизированные системы. Техническое задание на создание автоматизированной системы». Стандарт определяет требования к техническому заданию и разработке автоматизированных систем, обеспечивая единообразие подходов к проектированию.
:::

::: {custom-style="Normal"}
C4: Алгоритмы обработки заказов и платежей — бизнес-логика системы, определяющая последовательность действий при обработке заявок, расчете стоимости, проведении платежей, формировании документов. Алгоритмы определяют логику проверки доступности оборудования, расчет стоимости аренды, процесс оформления заказа, механизм проведения платежей.
:::

### Входящие потоки (Input)

::: {custom-style="Normal"}
Входящие потоки представляют собой информацию, которая поступает в систему извне и подлежит обработке.
:::

::: {custom-style="Normal"}
I1: Заявка клиента на аренду — запрос клиента на бронирование конкретного вида оборудования на определенный период времени. Заявка является основным входом, инициирующим работу системы. Она содержит информацию о желаемом оборудовании, датах аренды, дополнительных услугах.
:::

::: {custom-style="Normal"}
I2: Персональные данные клиента — личная информация клиента, необходимая для идентификации, регистрации и оформления договора аренды. Персональные данные необходимы для создания учетной записи клиента, оформления юридически значимых документов, связи с клиентом.
:::

::: {custom-style="Normal"}
I3: Данные об оборудовании — информация о спортивном инвентаре, имеющемся в наличии, его характеристиках, состоянии и местонахождении. Данные об оборудовании являются основой каталога системы и необходимы для отображения доступного инвентаря, проверки технического состояния, планирования обслуживания, учета движения на складе.
:::

::: {custom-style="Normal"}
I4: Платежная информация — данные о платежах клиентов, включая информацию о способе оплаты, сумме, статусе транзакции. Платежная информация критически важна для проведения финансовых транзакций, подтверждения оплаты услуг, формирования финансовой отчетности, обеспечения безопасности платежей.
:::

### Исходящие потоки (Output)

::: {custom-style="Normal"}
Исходящие потоки представляют собой результаты работы системы, которые передаются пользователям или внешним системам.
:::

::: {custom-style="Normal"}
O1: Договор аренды — официальный документ, фиксирующий условия аренды спортивного оборудования между компанией SportRent и клиентом. Договор является юридически значимым документом, который определяет права и обязанности сторон, устанавливает условия аренды и стоимость услуг, регулирует ответственность за ущерб или утрату оборудования.
:::

::: {custom-style="Normal"}
O2: Квитанция об оплате — документ, подтверждающий факт оплаты услуг аренды клиентом. Квитанция необходима для подтверждения совершения платежа, ведения финансового учета, предоставления клиенту доказательства оплаты, налоговой отчетности.
:::

::: {custom-style="Normal"}
O3: Акт приема-передачи оборудования — документ, фиксирующий состояние оборудования при выдаче клиенту и при его возврате. Акт приема-передачи служит для документирования состояния оборудования до и после аренды, фиксации комплектности инвентаря, разрешения споров о повреждениях.
:::

::: {custom-style="Normal"}
O4: Отчеты и аналитика — аналитические отчеты, содержащие статистику по работе системы, финансовым показателям, загрузке оборудования. Отчеты необходимы для анализа эффективности бизнес-процессов, принятия управленческих решений, планирования закупок нового оборудования, оценки финансовых результатов.
:::

::: {custom-style="Normal"}
O5: Информация о статусе заказа — актуальная информация о текущем состоянии заказа клиента (оформлен, подтвержден, готов к выдаче, выдан, завершен). Информация о статусе позволяет держать клиента в курсе обработки его заказа, снизить количество запросов в службу поддержки, повысить прозрачность процессов.
:::

### Механизмы (Mechanism)

::: {custom-style="Normal"}
Механизмы представляют собой ресурсы (люди, системы, оборудование), которые обеспечивают выполнение функций системы.
:::

::: {custom-style="Normal"}
M1: Клиенты — конечные пользователи системы, физические или юридические лица, арендующие спортивное оборудование. Клиенты являются основными пользователями системы, которые инициируют процесс аренды, взаимодействуют с веб-интерфейсом, осуществляют оплату услуг, получают и возвращают оборудование.
:::

::: {custom-style="Normal"}
M2: Менеджеры — бизнес-операторы, отвечающие за управление каталогом, ценообразование, обработку заказов и бизнес-аналитику. Менеджеры обеспечивают актуальность каталога оборудования, гибкое управление ценами, обработку нестандартных заказов, анализ бизнес-показателей.
:::

::: {custom-style="Normal"}
M3: Администраторы — системные администраторы с полным доступом к настройкам системы, управлению пользователями, безопасностью и техническому обслуживанию. Администраторы необходимы для обеспечения бесперебойной работы системы, управления безопасностью и правами доступа, настройки интеграций, мониторинга производительности.
:::

::: {custom-style="Normal"}
M4: Складской персонал — сотрудники, отвечающие за физические операции с оборудованием: выдачу клиентам, прием возвратов, проверку состояния, складской учет. Складской персонал обеспечивает физическую выдачу оборудования, контроль состояния инвентаря при возврате, ведение складского учета, подготовку оборудования к аренде.
:::

::: {custom-style="Normal"}
M5: СУБД (PostgreSQL/MySQL) — система управления базами данных, обеспечивающая надежное хранение, быстрый доступ и целостность всех данных системы. СУБД является критически важным компонентом, который обеспечивает персистентность всех данных, гарантирует ACID-свойства транзакций, поддерживает сложные запросы и аналитику.
:::

::: {custom-style="Normal"}
M6: Платежная система — внешний сервис обработки платежей (Stripe, PayPal, ЮKassa), интегрированный с системой SportRent для проведения финансовых транзакций. Платежная система необходима для безопасной обработки платежей клиентов, поддержки различных способов оплаты, соответствия стандартам безопасности.
:::

## Глоссарий

::: {custom-style="Normal"}
A-0 (контекстная диаграмма) — диаграмма верхнего уровня в нотации IDEF0, представляющая систему как единый функциональный блок с внешним окружением.
:::

::: {custom-style="Normal"}
ACID — набор свойств транзакций в СУБД: Atomicity (атомарность), Consistency (согласованность), Isolation (изолированность), Durability (долговечность).
:::

::: {custom-style="Normal"}
API (Application Programming Interface) — интерфейс программирования приложений, набор методов для взаимодействия между различными программными компонентами.
:::

::: {custom-style="Normal"}
HTTPS/TLS — протокол безопасной передачи данных через интернет с использованием шифрования.
:::

::: {custom-style="Normal"}
IDEF0 (Integration Definition for Function Modeling) — методология функционального моделирования, стандарт для создания функциональных моделей систем.
:::

::: {custom-style="Normal"}
JSON (JavaScript Object Notation) — текстовый формат обмена данными, основанный на JavaScript.
:::

::: {custom-style="Normal"}
RESTful API — архитектурный стиль для создания веб-сервисов, основанный на протоколе HTTP и принципах REST.
:::

::: {custom-style="Normal"}
SADT (Structured Analysis and Design Technique) — методология структурного анализа и проектирования систем.
:::

::: {custom-style="Normal"}
СУБД — система управления базами данных, программное обеспечение для создания, управления и доступа к базам данных.
:::

::: {custom-style="Normal"}
Акт приема-передачи — документ, фиксирующий факт передачи оборудования от одной стороны другой с описанием состояния и комплектности.
:::

::: {custom-style="Normal"}
Аутентификация — процесс проверки подлинности пользователя, обычно с использованием логина и пароля.
:::

::: {custom-style="Normal"}
Бронирование — предварительный заказ оборудования на определенный период времени.
:::

::: {custom-style="Normal"}
Декомпозиция — процесс разбиения сложной системы на более простые составные части для упрощения анализа и проектирования.
:::

::: {custom-style="Normal"}
Залог — денежная сумма, вносимая клиентом в качестве гарантии сохранности арендуемого оборудования.
:::

::: {custom-style="Normal"}
Информационная система (ИС) — совокупность технических и программных средств, персонала и процедур, предназначенных для сбора, хранения, обработки и передачи информации.
:::

::: {custom-style="Normal"}
Квитанция — документ, подтверждающий факт получения денежных средств.
:::

::: {custom-style="Normal"}
Модуль — функционально завершенная часть программной системы, выполняющая определенную задачу.
:::

::: {custom-style="Normal"}
Нормализация БД — процесс организации данных в базе данных для уменьшения избыточности и улучшения целостности.
:::

::: {custom-style="Normal"}
Платежный шлюз — сервис, обеспечивающий безопасную передачу платежной информации между клиентом, продавцом и банком.
:::

::: {custom-style="Normal"}
Складской учет — система контроля и документирования движения товарно-материальных ценностей на складе.
:::

::: {custom-style="Normal"}
Функциональный блок — элемент диаграммы IDEF0, представляющий функцию или процесс системы.
:::

## Вывод

::: {custom-style="Normal"}
В ходе выполнения практической работы была разработана контекстная диаграмма A-0 функциональной модели информационной системы SportRent в нотации IDEF0 с использованием методологии SADT.
:::

::: {custom-style="Normal"}
Созданная диаграмма наглядно представляет систему управления прокатом спортивного оборудования как единый функциональный блок с четко определенными входными потоками (заявки клиентов, персональные данные, информация об оборудовании и платежах), выходными потоками (договоры, квитанции, акты, отчеты и информация о статусе заказов), элементами управления (законодательство, политики компании, стандарты и алгоритмы обработки) и механизмами (пользователи различных ролей, СУБД и платежные системы).
:::

::: {custom-style="Normal"}
Разработанная модель соответствует требованиям руководящего документа РД IDEF0-2000 и может служить основой для дальнейшей декомпозиции системы на более детальные уровни представления.
:::

::: {custom-style="Normal"}
Применение методологии SADT и нотации IDEF0 позволило формализовать требования к информационной системе SportRent, выявить все основные информационные потоки и ресурсы, определить границы системы и её взаимодействие с внешним окружением, создать основу для дальнейшего проектирования и разработки системы.
:::

::: {custom-style="Normal"}
Полученная контекстная диаграмма может быть использована в качестве отправной точки для детальной декомпозиции функций системы на последующих уровнях моделирования, что позволит более подробно описать внутренние процессы каждого модуля системы SportRent.
:::
