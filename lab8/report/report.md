::: {custom-style="Heading 2"}
Описание элементарной семантической единицы (ЭСЕ)
:::

::: {custom-style="Normal"}
Элементарная семантическая единица (ЭСЕ) – неделимая единица информации, использующаяся в ИС. ЭСЕ представляет собой завершенную контекстную конструкцию, вызываемую в результате поиска по различным атрибутам или в результате тех или иных команд в виде отклика или отчета.
:::

::: {custom-style="Normal"}
В случае исследования системы SportRent за элементарную семантическую единицу была выбрана следующая характеристика: **количество заказов на аренду спортивного оборудования в день**. Эта величина меняется случайным образом в пределах от 50 до 150 заказов в день.
:::

::: {custom-style="Normal"}
Выбор данной ЭСЕ обусловлен следующими причинами:
:::

::: {custom-style="Normal"}
- Количество заказов является ключевым параметром для оценки нагрузки на систему
- Данный параметр варьируется в зависимости от сезона, дня недели, популярности оборудования
- Значение влияет на производительность системы и требования к ресурсам
- Параметр имеет измеримый диапазон значений, что позволяет провести статистический анализ
:::

::: {custom-style="Heading 2"}
Расчет математического ожидания информационного блока системы
:::

::: {custom-style="Normal"}
Математическим ожиданием случайной величины называется сумма произведений всех возможных значений случайной величины на вероятности этих значений. Рассчитаем математическое ожидание для нашей системы, взяв за случайную величину число заказов на аренду в день.
:::

::: {custom-style="Normal"}
Расчёт математического ожидания информационного блока на примере 100 записей выполняется по формуле:
:::

::: {custom-style="Normal"}
M(x) = Σ(x~i~ · P(x~i~))
:::

::: {custom-style="Normal"}
где x~i~ – центр i-го интервала, P(x~i~) – вероятность попадания значения в i-й интервал.
:::

::: {custom-style="Normal"}
Используя данные, полученные в таблице 2, получаем:
:::

::: {custom-style="Normal"}
M(x) = 55.0 · 0.09 + 65.0 · 0.13 + 75.0 · 0.14 + 85.0 · 0.10 + 95.0 · 0.11 + 105.0 · 0.06 + 115.0 · 0.05 + 125.0 · 0.09 + 135.0 · 0.13 + 145.0 · 0.10 = 98.20 заказов
:::

::: {custom-style="Normal"}
Следовательно, наиболее вероятное количество заказов на аренду находится в районе 98.20 заказов в день.
:::

::: {custom-style="Heading 2"}
Расчет дисперсии информационного блока системы
:::

::: {custom-style="Normal"}
Дисперсия характеризует меру разброса значений случайной величины относительно её математического ожидания. Расчёт дисперсии выполняется по формуле:
:::

::: {custom-style="Normal"}
D(x) = Σ((x~i~ - M(x))^2^ · P(x~i~))
:::

::: {custom-style="Normal"}
где x~i~ – центр i-го интервала, M(x) – математическое ожидание, P(x~i~) – вероятность попадания значения в i-й интервал.
:::

::: {custom-style="Normal"}
Используя данные, полученные в таблице 2 и вычисленное математическое ожидание M(x) = 98.20, получаем:
:::

::: {custom-style="Normal"}
D(x) = (55.0 - 98.20)^2^ · 0.09 + (65.0 - 98.20)^2^ · 0.13 + (75.0 - 98.20)^2^ · 0.14 + (85.0 - 98.20)^2^ · 0.10 + (95.0 - 98.20)^2^ · 0.11 + (105.0 - 98.20)^2^ · 0.06 + (115.0 - 98.20)^2^ · 0.05 + (125.0 - 98.20)^2^ · 0.09 + (135.0 - 98.20)^2^ · 0.13 + (145.0 - 98.20)^2^ · 0.10 = 881.76 заказов^2^
:::

::: {custom-style="Heading 2"}
Расчет среднеквадратического отклонения
:::

::: {custom-style="Normal"}
Среднеквадратическое отклонение характеризует меру разброса значений в тех же единицах измерения, что и исходные данные. Расчёт среднеквадратического отклонения выполняется по формуле:
:::

::: {custom-style="Normal"}
σ(x) = √D(x)
:::

::: {custom-style="Normal"}
Используя вычисленное значение дисперсии D(x) = 881.76, получаем:
:::

::: {custom-style="Normal"}
σ(x) = √881.76 = 29.69 заказов
:::

::: {custom-style="Normal"}
Это означает, что типичное отклонение количества заказов от среднего значения составляет около 29.69 заказов.
:::

::: {custom-style="Heading 2"}
Расчет энтропии системы
:::

::: {custom-style="Normal"}
Энтропия системы – это сумма произведений вероятностей различных состояний системы на логарифмы этих вероятностей, взятая с обратным знаком. Энтропия характеризует степень неопределенности информации в системе.
:::

::: {custom-style="Normal"}
Расчёт энтропии выполняется по формуле Шеннона:
:::

::: {custom-style="Normal"}
H(x) = -Σ(P(x~i~) · log~2~(P(x~i~)))
:::

::: {custom-style="Normal"}
где P(x~i~) – вероятность i-го состояния системы, log~2~ – логарифм по основанию 2.
:::

::: {custom-style="Normal"}
За основание логарифма взята двоичная система счисления. Энтропия фрагмента информационного наполнения в размере 100 ЭСЕ:
:::

::: {custom-style="Normal"}
Используя данные, полученные в таблице 2, получаем:
:::

::: {custom-style="Normal"}
H(x) = -(0.09 · log~2~(0.09) + 0.13 · log~2~(0.13) + 0.14 · log~2~(0.14) + 0.10 · log~2~(0.10) + 0.11 · log~2~(0.11) + 0.06 · log~2~(0.06) + 0.05 · log~2~(0.05) + 0.09 · log~2~(0.09) + 0.13 · log~2~(0.13) + 0.10 · log~2~(0.10)) = 3.262 бит
:::

::: {custom-style="Normal"}
Значение энтропии 3.262 бит указывает на достаточно высокую степень неопределенности в системе, что свидетельствует о разнообразии возможных состояний системы.

Контрольные вопросы
:::

::: {custom-style="Normal"}
**1. Что такое энтропия информационной системы?**
:::

::: {custom-style="Normal"}
Энтропия информационной системы – это мера неопределенности или степень хаотичности информации в системе. Она количественно характеризует среднее количество информации, содержащееся в сообщении о состоянии системы. Чем выше энтропия, тем более непредсказуемым является состояние системы, и тем больше информации требуется для его описания.
:::

::: {custom-style="Normal"}
**2. Дайте определение термину «Семантическая информация».**
:::

::: {custom-style="Normal"}
Семантическая информация – это информация, содержащая смысловое содержание сообщения. Семантическая мера информации связана с тезаурусом пользователя и отражает смысловую ценность информации для получателя. Она учитывает не только количество данных, но и их полезность и значимость для решения конкретных задач.
:::

::: {custom-style="Normal"}
**3. Что такое энтропия Шеннона?**
:::

::: {custom-style="Normal"}
Энтропия Шеннона – это математическая мера неопределенности случайной величины, введенная Клодом Шенноном в 1948 году. Она вычисляется по формуле H(x) = -Σ(P(x~i~) · log~2~(P(x~i~))), где P(x~i~) – вероятность i-го состояния системы. Энтропия Шеннона измеряется в битах и показывает среднее количество информации, необходимое для описания состояния системы.
:::

::: {custom-style="Normal"}
**4. Что такое дисперсия информации?**
:::

::: {custom-style="Normal"}
Дисперсия информации – это статистическая мера разброса значений случайной величины относительно её математического ожидания. Дисперсия вычисляется как среднее значение квадратов отклонений от математического ожидания: D(x) = Σ((x~i~ - M(x))^2^ · P(x~i~)). Чем больше дисперсия, тем более разнообразны значения величины.
:::

::: {custom-style="Normal"}
**5. Взаимосвязь реляционных данных, корреляция данных.**
:::

::: {custom-style="Normal"}
Взаимосвязь реляционных данных характеризует зависимость между различными атрибутами и таблицами в базе данных. Корреляция данных – это статистическая мера линейной зависимости между двумя переменными, принимающая значения от -1 до +1. Положительная корреляция означает, что при увеличении одной переменной увеличивается другая, отрицательная – что при увеличении одной уменьшается другая. Нулевая корреляция указывает на отсутствие линейной зависимости.
:::

::: {custom-style="Normal"}
**6. Что такое модели данных?**
:::

::: {custom-style="Normal"}
Модели данных – это формализованные способы описания и организации данных, их связей, ограничений целостности и операций над данными. Существуют различные типы моделей данных: реляционная (таблицы с связями), иерархическая (древовидная структура), сетевая (граф), объектно-ориентированная. Модель данных определяет структуру базы данных и правила работы с информацией.
:::

::: {custom-style="Normal"}
**7. Что такое мера информации?**
:::

::: {custom-style="Normal"}
Мера информации – это количественная характеристика информации. Существуют различные меры информации: синтаксическая (объем данных в битах или байтах), семантическая (смысловое содержание), прагматическая (полезность для достижения цели). Основной мерой количества информации является бит – минимальная единица информации, соответствующая выбору из двух равновероятных альтернатив.
:::

::: {custom-style="Normal"}
**8. Как выполнить анализ параметров проектируемой информационной системы?**
:::

::: {custom-style="Normal"}
Анализ параметров проектируемой информационной системы выполняется в следующей последовательности:
:::

::: {custom-style="Normal"}
1. Определение элементарной семантической единицы (ЭСЕ) – ключевого параметра системы
2. Сбор статистических данных о значениях ЭСЕ
3. Построение ряда распределения вероятностей
4. Расчет математического ожидания (среднее значение параметра)
5. Расчет дисперсии и среднеквадратического отклонения (разброс значений)
6. Расчет энтропии системы по формуле Шеннона (степень неопределенности)
7. Интерпретация полученных результатов для принятия проектных решений
:::
